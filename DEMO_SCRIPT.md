### 二、題目介紹

**I'm cooked** 是一個食譜分享平台。使用者可以在這邊上傳自己的食譜，也可以瀏覽和收藏其他人的食譜，還有評分和留言功能。此外，我們特別做了一個烹飪模式，讓使用者可以一步步跟著食譜做菜，另外還有搜尋、標籤篩選這些功能，讓大家更容易找到想做的料理。

### 三、程式碼架構與使用技術介紹

在技術方面，我們主要用了幾個工具：

**前端**我們用 **Next.js 14** 搭配 **TypeScript** 來開發，這樣可以確保程式碼的品質和類型安全。我們用 Next.js 的 App Router 來管理路由，每個資料夾就是一個頁面。樣式部分用 **Tailwind CSS** 來做，讓網站可以同時支援手機和電腦。表單部分我們有做驗證，確保使用者填的資料格式正確。

**後端**我們用 **Supabase**，它幫我們處理資料庫和使用者認證。資料庫用的是 PostgreSQL，我們設計了幾個主要的資料表，像是使用者資料、食譜、評分、收藏、留言這些。食譜的食材和步驟我們用 JSON 格式存在資料庫裡，這樣可以把整個陣列直接存起來，比較有彈性。我們還用了資料庫觸發器來自動更新一些統計資訊，像是評分數量、收藏數量這些，這樣就不用每次都重新計算。為了安全，我們有設定 Row Level Security，確保每個使用者只能看到和修改自己的資料。

**資料處理**我們用 Next.js 的 Server Actions 來處理資料操作，這樣敏感的操作都在伺服器端執行，比較安全。

**圖片處理**我們用 **Cloudinary** 來上傳和儲存食譜的圖片，它會自動幫我們優化圖片大小。

**部署**方面，我們把網站放在 **Vercel** 上面，每次更新程式碼就會自動部署，很方便。

整體來說，我們把程式碼分成幾個模組，像是 `app` 目錄放頁面、`components` 目錄放可重用的元件、`lib` 目錄放資料處理邏輯，這樣程式碼比較好維護。
